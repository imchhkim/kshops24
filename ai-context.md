## KShops24 개발 지침
1. 대상: 전세계 거주 한국인 소상공인 (모바일 이용자 99%)
2. 환경: 모바일 웹 사이트가 중요
3. 인증: 카카오 로그인 필수
4. 기술 스택: PHP, MySQL, Bootstrap 5, Kakao JavaScript SDK
5. 핵심 개발 원칙 (매우 중요):
   - 개발 환경(URL : test.kshops24.com, 서버 폴더 : /public_html/test_env)에서 개발 작업을 한 후에, git을 이용해서 서비스 환경(URL : kshops24.com, 서버 폴더 : /public_html)로 코드가 이동되는 구조.
   - 땜빵식(Band-aid) 코딩 절대 금지: 버그 발생 시 억지로 이벤트를 막거나 덮어씌우지 말고, 프레임워크와 네이티브 로직의 근본 원인을 파악해 구조적으로 해결한다.
   - 모듈화 및 리팩토링 지향 (DRY 원칙): 코드 작성 시 항상 기능의 재사용성을 고려한다. 중복 코드를 방지하고 공통 파일(/common/lib_utils.php 등)을 활용해 확장성 높은 모듈 형태로 코딩한다.
   - 전체 시스템의 파일 구조를 나타내는 파일(PROJECT_STRUCTURE.md), DB table의 구조(DDL)을 보여주는 파일(schema.sql)을 참고하여 코딩한다.
   - 파일 절대 경로와 포맷에 맞춘 완벽한 Diff 코드를 제시하는 방식으로 작업을 수행한다.
6. 특이사항: 코드를 신규로 작성하거나 수정할 때, 반드시 자세한 코멘트를 함께 넣어 주세요. CSS를 제작할 때 중복 정의를 피하고, `!important` 선언을 가능하면 사용하지 않습니다. 각 설정 섹션(카드)에 고유한 이름(ID 앵커)을 부여하고, 데이터를 전송(submit)하거나 화면을 갱신(reload)할 때 화면 깜빡임을 최소화 시키는 AJAX 기술을 가능하면 사용한다. definded구문 사용을 자제하고, 상수를 정의해야 하는 경우는 /config.php 파일이나 부모 파일에 상수를 정의하고, 이를 사용합니다. CRUD 작업에서, 리스는 페이징 처리 해주고, 기본적인 검색 기능을 넣어준다. 함수를 만들어 사용할 때는, 이 함수의 재사용 성을 판단해서, 향후 다른 곳에서도 사용될 것 같다면, /common/lib_utils.php에 기록하고 재사용 한다.
